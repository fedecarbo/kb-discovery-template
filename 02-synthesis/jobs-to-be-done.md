## How this fits in the evidence chain

```
insights → JTBD → story map → opportunities → decisions
```

- **Insights** (`02-synthesis/insights/`) are atomic, session-grounded claims about what users experience.
- **JTBD** (this file) translates those claims into goals: the *why* behind the behaviour. One job can draw on multiple insights; one insight can feed multiple jobs.
- **Story map** (`02-synthesis/story-map.md`) is built *from* JTBD entries: it sequences jobs into activities and steps for a solution. Never build the story map directly from insights, bypassing this layer.
- **Opportunities** (`02-synthesis/opportunities.md`) are the design openings that arise when a job is poorly served.

Every JTBD entry **must cite ≥1 insight ID or session ID**. If a job can't be traced back, mark it `[ASSUMPTION]`: it's a hypothesis that needs research, not a finding.
